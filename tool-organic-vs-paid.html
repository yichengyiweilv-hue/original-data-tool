自然量 & 买量 Excel → JS 源数据转换工具
上传 Excel 后，点击「生成 code」，得到可以直接粘贴到你看板数据源的 "YYYY-MM": [ { month: "YYYY-MM", ... } ] 片段。
· 自然量：默认读取 Excel 的第一个 sheet，首行为英文表头（date / country / registration / 各类 D0 / D7 字段），下面每行是「日期 × 国家」数据。
· 买量：同样默认读取第一个 sheet，首行为英文表头（date / country / media / Product_type / spent / 各类 D0 / D7 字段）。
· 输出示例："2025-10": [ { month: "2025-10", date: "2025-10-01", country: "GH", ... } ]。
白色系主题 · 纯前端 · 无需安装软件 输出字段对齐你当前看板模版
自然量 Excel → JS 片段
输出格式："YYYY-MM": [ { month, date, country, regs, d0Payers, d7Payers, ... } ]， 直接粘贴到自然量月度看板的数据对象。
自然量 Excel：
· 要求：首行为英文表头，至少包含 date / country / registration / D0_unique_purchase / D7_unique_purchase / D0_SPORTS_GGR_VALUE / D7_SPORTS_GGR_VALUE / D0_GAMES_GGR_VALUE / D7_GAMES_GGR_VALUE / D1_retained_users / D7_retained_users 等列。
· retentionD1 / retentionD7 = 对应留存人数 ÷ registration，输出为小数（如 0.35）。
买量 Excel → JS 片段
输出格式： "YYYY-MM": [ { month, date, country, media, productType, spend, ... } ]， 直接粘贴到买量月度看板的数据对象。
买量 Excel：
· 要求：首行为英文表头，至少包含 date / country / media / Product_type / spent / registration / D0_unique_purchase / D7_unique_purchase / D0_SPORTS_GGR_VALUE / D7_SPORTS_GGR_VALUE / D0_GAMES_GGR_VALUE / D7_GAMES_GGR_VALUE / D1_retained_users / D7_retained_users 等列。
· spend 来自 spent 列，留存率计算逻辑同自然量。
使用建议：每月导出自然量 / 买量 Excel 后，用本页面生成对应月份 JS 片段，直接粘贴到各看板的数据源区域。
